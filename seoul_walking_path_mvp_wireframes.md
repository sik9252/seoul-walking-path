# 1. 문서 목적

이 문서는 **서울 전용 산책/걷기 앱 MVP**를 Stitch(디자인 생성/구성 도구)가 읽고 UI를 설계할 수 있도록, 화면별 구조를 **일관된 Markdown 규격**으로 정리한 와이어프레임 스펙입니다.

- 제품 핵심 가치:  
  1) 서울 산책 코스 탐색(거리/시간/난이도)  
  2) GPS 트래킹 기반 산책 중 실시간 안내(코스 라인 + 주변시설 POI)

---

# 2. MVP 범위 요약

## 2.1 포함 기능 (MVP)
- 코스 탐색: 추천/리스트/검색/필터/정렬
- 코스 상세: 거리/시간/난이도/지도 프리뷰/시설 요약
- 산책 트래킹: 시작/일시정지/종료, 거리/시간/트랙 기록
- 산책 중 지도: 코스 라인 + 내 위치 + 내 트랙 + POI 표시(5종)
- 산책 기록: 목록/상세(지도 + 통계)
- 즐겨찾기(코스 저장)

## 2.2 제외(후순위)
- 보상/스탬프/챌린지/리그
- 소셜 피드/팔로우
- 사용자 제작 코스(GPX 업로드 등)
- 고급 건강 데이터 분석/코칭

# 3. IA / 내비게이션 구조

## 3.1 Bottom Tab (고정 4탭)
- 홈 (Home)
- 코스 (Routes)
- 기록 (Records)
- 마이 (My)

> 트래킹 중(S7)에는 바텀탭 숨김(몰입/오조작 방지). 종료 요약(S8)부터 재노출.

## 3.2 핵심 사용자 플로우
1) 스플래시(S0) → 온보딩(S1) → 위치 권한(S2) → 홈(S3)  
2) 홈/코스(S3/S4) → 코스 상세(S5) → 산책 시작 전 체크(S6) → 트래킹(S7) → 종료 요약(S8) → 기록(S9/S10)

# 4. 전역 UI 패턴 (Global Patterns)

## 4.1 App Bar (상단)
- 좌: 뒤로가기(필요 시)
- 중앙: 화면 타이틀
- 우: (화면별) 검색/필터/정렬/설정/즐겨찾기

## 4.2 지도(Map) 공통 패턴
- 지도 위에 Bottom Sheet로 정보/액션 제공
- 우하단 FAB: “내 위치로 이동(센터링)”
- 줌 컨트롤(+/-)
- 지도 로딩 실패 시: 재시도 버튼 제공(또는 대체 뷰)

## 4.3 알림/상태 패턴
- Top Banner(비침투형): GPS 정확도 낮음, 백그라운드 제한, 경로 이탈 등
- Toast: 즐겨찾기 추가/삭제, 저장 완료, 필터 적용
- Full Screen Empty: 권한 거부/데이터 없음/오프라인

# 5. 컴포넌트 라이브러리 (MVP)

## 5.1 공통 컴포넌트
- `RouteCard` : 홈 추천 코스 가로 카드
- `RouteListItem` : 코스 리스트 아이템
- `FilterChip` / `TagChip` : 필터/태그 칩
- `MapPreview` : 코스 상세 상단 지도 프리뷰(폴리라인)
- `BottomSheet` : 지도 위 정보 패널(접힘/확장)
- `TrackingHUD` : 트래킹 통계(시간/거리/페이스) + 컨트롤
- `POICategoryChips` : POI 카테고리 토글(5종)
- `POICard` : 지도에서 POI 탭 시 미니 카드
- `RecordCard` : 기록 리스트 카드
- `RecordMapThumb` : 기록 썸네일 미니맵
- `TopBanner` : 상태 배너
- `PrimaryButton` / `SecondaryButton` / `DestructiveButton`

## 5.2 POI 카테고리 (MVP 고정 5종)
1) 화장실  
2) 편의점  
3) 카페  
4) 지하철역  
5) 버스정류장 **또는** 공영주차장(1개 선택)

# 6. 화면 스펙 (Screen-by-Screen)

> 표기 규칙  
> - Screen ID: `S#`  
> - 각 화면은 **목적 / 진입 / 레이아웃 / 인터랙션 / 상태**로 구성  
> - 레이아웃은 “상→하” 순서로 적습니다.

---

## S0 — 스플래시 (Splash)

### 목적
초기 로딩(코스 데이터 캐시/원격 설정 등) 및 앱 진입 준비.

### 진입
앱 실행 시 항상 진입.

### 레이아웃
- 로고(중앙)
- 로딩 인디케이터
- (2초 이상) “코스 데이터 준비 중…” 보조 문구

### 상태
- 로딩 실패: “네트워크 상태를 확인해주세요” + `재시도`

---

## S1 — 온보딩 (Onboarding)

### 목적
앱 가치 전달 + 위치 권한 허용 동기 제공(강요 X).

### 진입
최초 1회(또는 로그아웃/초기화 시).

### 레이아웃(페이지형)
- 상단: `Skip`
- 중앙: 일러스트/아이콘 + 헤드라인 + 서브 텍스트
- 하단: 페이지 도트 + `다음`/`시작하기`

### 페이지 카피(예시)
1) “서울 산책 코스를 한눈에” / “거리·시간·난이도 기준으로 골라요”
2) “산책 중 코스와 주변시설을 지도에서”
3) “GPS로 산책 기록을 남겨요” + `시작하기`

---

## S2 — 위치 권한 안내 (Permission Explainer)

### 목적
위치 권한 요청을 명확히 설명하고 OS 권한 팝업으로 연결.

### 진입
- 온보딩 완료 후
- 또는 권한 거부 후 재진입

### 레이아웃
- 타이틀: “위치 권한이 필요해요”
- 설명: “내 주변 코스 추천과 산책 기록을 위해 사용합니다”
- 불릿: 내 주변 코스 / 산책 경로 기록 / 주변시설 안내
- CTA:
  - `위치 권한 허용` (Primary)
  - `나중에` (Secondary)

### 상태: 권한 거부(S2-1)
- 타이틀: “위치 권한이 꺼져 있어요”
- 설명: “코스 탐색은 가능하지만 내 주변/기록 기능이 제한됩니다.”
- CTA: `설정으로 이동`, `코스 둘러보기`

---

## S3 — 홈 (Home / Tab)

### 목적
빠른 탐색/시작(내 주변 추천) + 테마 추천 + 최근 활동 노출.

### 진입
앱 메인 진입 탭.

### 레이아웃(상→하)
1) App Bar: 타이틀(앱명) + `검색(아이콘)` + `설정(아이콘)`
2) 현재 위치 스트립: “OO동 · 업데이트(⟳)”
3) 퀵 필터 칩: `30분` `60분` `1~3km` `한강` `공원` `숲길` `더보기`
4) 섹션 A: “내 주변 추천” (가로 스크롤 `RouteCard`)
5) 섹션 B: “오늘의 테마” (테마 카드)
6) 섹션 C: “최근 산책”(있을 때만) + 기록 탭 딥링크

### 인터랙션
- `RouteCard` 탭 → S5(코스 상세)
- 퀵 필터 탭 → 홈 추천 리스트에 즉시 반영(또는 코스탭으로 이동)
- 검색 아이콘 → S4(코스 목록/검색 포커스)

### 상태
- 위치 권한 없음: “내 주변 추천” 섹션에 권한 안내 카드 + `권한 켜기`
- 로딩: 코스 카드 스켈레톤
- 네트워크 실패: 캐시 fallback + 상단 배너

---

## S4 — 코스 목록 (Routes List / Tab)

### 목적
검색/필터/정렬로 코스를 탐색하고 비교.

### 진입
- 바텀탭 코스
- 홈에서 검색/더보기/테마 진입

### 레이아웃(상→하)
1) App Bar: “코스” + `검색` `필터` `정렬`
2) 검색 입력(옵션): “지역/코스명을 검색하세요”
3) 활성 필터 요약 바: “60분 · 1~3km · 한강 · 성동구” + `초기화`
4) 코스 리스트: 세로 `RouteListItem`

### 인터랙션
- `RouteListItem` 탭 → S5(코스 상세)
- 정렬 버튼 → S4-1(정렬 Bottom Sheet)
- 필터 버튼 → S4-2(필터 Bottom Sheet)

### 상태
- 결과 없음: “조건에 맞는 코스가 없어요” + `필터 초기화`
- 위치 권한 없음: “가까운순” 비활성/안내 문구

#### S4-1 — 정렬 Bottom Sheet
- 추천순(기본), 가까운순, 거리 짧은순, 시간 짧은순

#### S4-2 — 필터 Bottom Sheet
- 거리: 1~3 / 3~6 / 6+
- 시간: 30 / 60 / 90+
- 지역(구): 멀티 선택
- 테마: 한강/공원/숲길/야경/역세권…
- CTA: `적용`, `초기화`

---

## S5 — 코스 상세 (Route Detail)

### 목적
코스 선택 확정 + 산책 시작.

### 진입
홈/코스 리스트에서 코스 선택.

### 레이아웃(상→하)
1) App Bar: 뒤로 + 코스명 + `즐겨찾기(♡)`
2) 상단 지도 프리뷰(`MapPreview`)
   - 코스 폴리라인
   - 시작/종료 마커
   - 내 위치(가능 시)
3) 핵심 메타 블록
   - 거리(km) · 예상시간(분) · 난이도 배지
   - 태그 칩 1~3개
4) 주요 포인트(옵션)
5) 주변 시설 요약(카운트)
   - 화장실/편의점/카페/지하철/버스(또는 주차)
   - “지도에서 보기” 토글(아이콘 표시 on/off)
6) 주의사항/안내(고정 문구 가능)
7) 하단 CTA
   - `산책 시작` (Primary)
   - `시작점 길찾기` (Secondary)
   - `정보 제보` (텍스트 링크)

### 인터랙션
- ♡ 토글: 즐겨찾기 추가/삭제 Toast
- `산책 시작` → S6(체크) → S7(트래킹)

### 상태
- 지도 로딩 실패: 재시도 + 대체 텍스트
- 데이터 불완전: “예상 시간은 참고용” 안내

---

## S6 — 산책 시작 전 체크 (Pre-Start Checks)

### 목적
트래킹 실패 예방(권한/백그라운드/시작점 이격 등).

### 진입
S5에서 `산책 시작` 누른 직후.

### 분기 A: 위치 권한 없음/불충분
- 타이틀: “산책 기록을 위해 위치 권한이 필요해요”
- CTA: `권한 허용`, `취소`

### 분기 B: 백그라운드 권한 권장(선택)
- 타이틀: “화면이 꺼져도 기록하려면 ‘항상 허용’이 좋아요”
- CTA: `설정 열기`, `일단 시작`

### 분기 C: 시작점과 거리 있음(예: 300m+)
- 타이틀: “시작점에서 조금 떨어져 있어요”
- CTA: `시작점으로 길찾기`, `현재 위치에서 시작`

---

## S7 — 산책 트래킹 (Tracking / Core)

### 목적
GPS 트래킹 + 코스/POI 실시간 안내.

### 진입
S6 체크 완료 후.

### 레이아웃(상→하)
1) 상단 미니바
   - 좌: 코스명(또는 “자유 산책”)
   - 우: GPS 상태(좋음/낮음)
2) 메인 지도
   - 코스 폴리라인(선택 코스일 때)
   - 내 트랙(실제 걸은 경로)
   - 내 위치 화살표(방향)
   - POI 아이콘(카테고리별)
3) 우측/우하단 컨트롤
   - 내 위치 센터링 FAB
   - 줌 +/-
   - 레이어/POI 토글
4) 하단 Bottom Sheet(기본 접힘)
   - 시간 / 거리 / 평균 페이스(선택)
   - 버튼: `일시정지`(Primary), `종료`(Destructive)

### POI UX
- `POICategoryChips`: [화장실] [편의점] [카페] [지하철] [버스/주차]
- 지도에서 POI 탭 → `POICard`:
  - 이름, 현재 위치 기준 거리, `길찾기(외부 지도)`

### 인터랙션
- `일시정지` → S7-1(오버레이)
- `종료` → S8(종료 요약)
- POI 카테고리 토글: 지도 마커 필터링
- 센터링: 내 위치 기준으로 지도 이동

### 상태/배너
- GPS 정확도 낮음 배너: “건물/터널 주변에서는 오차가 생길 수 있어요”
- (P1) 경로 이탈 배너: “코스에서 벗어났어요 · 코스로 돌아가기”
- 백그라운드 중단 감지 배너: “기록이 일시 중단됐어요 · 설정 확인”

#### S7-1 — 일시정지 오버레이
- 큰 버튼 중심:
  - `재개` (Primary)
  - `종료하고 저장`
  - `기록 폐기` (Destructive, 2차 확인 다이얼로그)

---

## S8 — 산책 종료 요약 (Walk Summary)

### 목적
성취감 제공 + 기록 저장/삭제 결정.

### 진입
S7에서 `종료`.

### 레이아웃(상→하)
1) 상단 지도 스냅샷(트랙/코스 표시)
2) 요약 카드
   - 총 거리, 총 시간
   - 평균 페이스(선택)
   - (코스 연결 시) 완료율/완료 여부(가능할 때만)
3) 메모 입력(선택): “오늘 산책은 어땠나요?”
4) CTA
   - `저장` (Primary)
   - `삭제` (Destructive)

### 저장 후
- Toast: “기록이 저장됐어요”
- `기록 보기` → S10(기록 상세) 또는 S9(기록 목록)

---

## S9 — 기록 목록 (Records List / Tab)

### 목적
산책 기록을 날짜 기준으로 확인, 재방문 유도.

### 진입
바텀탭 기록 / 저장 완료 후 딥링크.

### 레이아웃(상→하)
1) App Bar: “기록” + (정렬)
2) (선택) 이번 주 요약 스트립: 총 km/횟수
3) 날짜 그룹 헤더 + `RecordCard` 리스트

### 상태
- 기록 없음:
  - “아직 기록이 없어요”
  - CTA: `코스 보러가기` → S4

---

## S10 — 기록 상세 (Record Detail)

### 목적
기록 1건의 지도/통계 확인, 코스 재탐색.

### 진입
S9에서 `RecordCard` 탭.

### 레이아웃(상→하)
1) App Bar: 뒤로 + “기록 상세” + (⋯)
2) 상단 지도
   - 트랙 폴리라인
   - (코스 연결 시) 코스 라인 약하게 표시
3) 통계 블록
   - 거리 / 시간 / 평균 페이스 / 시작 시각
4) 연결된 코스(있을 때만)
   - 코스명 + `코스 상세로` → S5
5) 삭제(옵션)
   - (⋯ 메뉴 또는 하단) `삭제`(Destructive)

## S11 — 마이/설정 (My / Settings / Tab)

### 목적
트래킹 설정/권한/데이터/지원.

### 레이아웃(상→하)
1) 트래킹 설정
   - 위치 정확도: `균형` / `정확` (세그먼트)
   - POI 표시: On/Off
   - 자동 화면 켜짐 유지(산책 중): On/Off
2) 데이터/개인정보
   - 내 산책 기록 전체 삭제
   - 위치 권한 설정으로 이동
   - 개인정보 처리방침 / 이용약관
3) 고객지원
   - 문의하기(메일/링크)
   - 앱 버전

## S13 — 정보 제보 (Report Issue) (P2)

### 목적
코스/시설 데이터 품질 개선 루프 구축(초기 운영에 유용).

### 진입
S5(코스 상세) 하단 링크 또는 S8(요약)에서 옵션.

### 레이아웃(상→하)

2) 제보 대상(자동 표시): 코스명 / (선택) 세션

   - 코스 정보가 달라요
   - 시설이 없어요
   - 기타

5) CTA: `제출`

### 제출 완료
- “제보가 접수됐어요. 감사합니다.” + 닫기

# 9. 디자인 가이드(간단)

- 트래킹(S7)은 **한 손 조작** 우선: 버튼 크기 크게, 하단 집중.
- 지도 위 정보는 Bottom Sheet로 제공(지도 가림 최소화).
- 위험 액션(삭제/폐기)은 항상 2차 확인.
- “권한/배터리” 안내는 과도한 방해 없이, 필요한 순간에만 노출.
